# 调试场景点击手牌出牌设计

## 背景

当前 Godot 调试场景使用固定的 `StrikeButton` 和 `DefendButton` 触发出牌。这适合早期冒烟测试，但和 Python 参考项目的流程不完全一致。Python 参考项目会从当前手牌生成可选项，选择某张牌后进入 `PlayCardAction`，如果牌需要敌人目标，再处理目标选择。

本设计把调试场景改为“点击手牌里的卡牌出牌”。固定按钮只保留结束回合、重开战斗、应用数值、详细日志等调试控制。

## 目标

- 固定调试战斗牌组包含：`打击`、`防御`、`打击`、`防御`、`痛击`、`燃烧`、`耸肩无视`。
- 五张测试卡显示中文名：
  - `Strike` 显示为 `打击`
  - `Defend` 显示为 `防御`
  - `Bash` 显示为 `痛击`
  - `Inflame` 显示为 `燃烧`
  - `Shrug It Off` 显示为 `耸肩无视`
- 调试场景根据当前手牌动态生成可点击卡牌按钮。
- 点击手牌按钮时，打出被点击的那一张牌，而不是按名字寻找第一张牌。
- 敌人目标牌在当前单敌人调试战斗中默认选第一个存活敌人。
- 自身目标牌和无目标牌直接结算。
- 出牌后刷新血量、能量、格挡、状态效果、手牌、抽牌堆、弃牌堆和日志。

## 非目标

- 不制作正式卡牌 UI、拖拽手感、目标箭头或多敌人目标选择界面。
- 不引入正式卡牌库、资源化内容编辑器或消息总线。
- 不改动 `slay-the-model-main/` Python 参考项目。
- 不把测试卡迁移出 `scripts/stm/cards/test/`。
- 不改变 `StmCombat.play_card()` 的核心结算职责。

## 设计

### 调试牌组

`StmFixedBattleFixture.create_deck()` 继续作为调试战斗的唯一牌组入口。它会 preload 五张测试卡脚本并返回七张卡实例。这样调试战斗、测试和手动体验使用同一份牌组。

### 中文卡牌名

五张测试卡的 `card_name` 改为中文显示名。现阶段 `card_name` 同时承担显示和测试调试查找用途，因此调试出牌逻辑也改为直接持有卡牌引用，避免中文名变化导致固定按钮查找不稳定。

### 手牌按钮

调试场景新增一个手牌按钮容器，例如 `HandButtons`。每次 `_refresh_display()` 时根据 `player.card_manager.get_pile("hand")` 重建按钮：

- 每张手牌一个按钮。
- 按钮文本包含卡牌名和费用，例如 `痛击（2）`。
- 按钮绑定对应卡牌引用，点击后调用 `_play_card_from_hand(card)`。
- 能量不足、战斗未开始、敌人目标不存在时禁用对应按钮。

原有 `HandLabel` 继续保留，用于显示手牌摘要，方便策划同时看列表和点击入口。

### 出牌流程

`_play_card_from_hand(card)` 负责调试 UI 层的输入转换：

1. 检查 `game_state`、`combat`、`card` 是否有效。
2. 如果卡牌不在当前手牌中，显示失败日志并刷新。
3. 根据 `target_type` 构造目标数组：
   - `enemy_select` 默认选择第一个存活敌人。
   - `self` 或其他无敌人选择需求传空数组。
4. 调用现有 `combat.play_card(game_state, card, targets)`。
5. 记录简洁日志和详细日志。
6. 刷新显示。

这保持了“调试 UI 只选择动作入口，战斗系统负责结算”的边界。

### 旧固定按钮

`StrikeButton` 和 `DefendButton` 不再作为主要出牌入口。为了避免旧测试路径绑死正式结构，新的测试应以 `HandButtons` 为核心。旧节点可以在本次直接移除，或者保留但隐藏；推荐直接移除固定出牌按钮，让调试界面行为更清楚。

## 安全、边界、依赖

- 安全：所有点击入口都检查战斗上下文、卡牌仍在手牌、敌人目标存在、能量是否足够。
- 边界：调试场景只读手牌并调用 `combat.play_card()`，不复制卡牌结算规则。
- 依赖：只依赖 Godot 4.6.2、GDScript、现有 GUT、现有 `StmCombat` / `StmCard` / `StmFixedBattleFixture`。
- 资源元数据：新增测试卡已有 Godot `.gd.uid` 文件时，应随本功能提交，保持脚本资源引用稳定。

## 测试策略

- 在 `test_fixed_battle_fixture.gd` 中验证固定调试牌组包含七张中文名卡牌。
- 在 `test_powers_v1.gd` 中把三张状态测试卡断言更新为中文名。
- 在 `test_battle_debug_scene.gd` 中验证初始手牌区域生成可点击卡牌按钮。
- 在 `test_battle_debug_scene.gd` 中验证点击 `痛击` 会造成伤害并给敌人易伤。
- 在 `test_battle_debug_scene.gd` 中验证点击 `燃烧` 会给玩家力量。
- 在 `test_battle_debug_scene.gd` 中验证点击 `耸肩无视` 会获得格挡并抽牌。
- 运行完整 GUT：`godot -s addons/gut/gut_cmdln.gd`。

## 验收

- 调试场景不再依赖固定 `Strike` / `Defend` 出牌按钮。
- 策划可以通过点击当前手牌中的卡牌完成出牌。
- 新增三张测试卡出现在固定调试战斗中，并能被手动点击验证。
- 中文卡名贯穿牌组、手牌摘要、按钮和日志。
- 完整 GUT 通过。
